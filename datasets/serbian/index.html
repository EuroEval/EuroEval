    This is an overview of all the datasets used in the Serbian part of EuroEval. The datasets are grouped by their task - see the task overview for more information about what these constitute.

    Sentiment Classification

    MMS-sr

    This dataset was published in this paper. The corpus consists of 79 manually selected datasets from over 350 datasets reported in the scientific literature based on strict quality criteria.

    The original dataset contains a single split with 6,165,262 samples. We use 1,024 / 256 / 2,048 samples for our training, validation and test splits, respectively.

    Here are a few examples from the training split:

    {
    "text": "Primiti manje od 10 trojki je uspeh za Radonjica.",
    "label": "neutral"
}

    {
    "text": "RT @Susanna_SQ: Osecati se dobro u sopstvenoj kozi, mozda je jedna od najvecih umetnosti zivljenja.",
    "label": "positive"
}

    {
    "text": "RT @aleksitimija_: ljubav je prolazna. prijatelji su prolazni. strast je prolazna. sve je prolazno. jedino sto mi ostaje jeste puno misli i...",
    "label": "negative"
}

    When evaluating generative models, we use the following setup (see the methodology for more information on how these are used):

    • Number of few-shot examples: 12
    • Prefix prompt:
    U nastavku su dokumenti i njihov sentiment, koji mo쬰 biti 'pozitivan', 'neutralan' ili 'negativan'.
    • Base prompt template:
    Dokument: {text}
Sentiment: {label}
    • Instruction-tuned prompt template:
    Dokument: {text}

Klasifikujte sentiment u dokumentu. Odgovorite sa 'pozitivan', 'neutralan', ili 'negativan', i ni코ta drugo.
    • Label mapping:
    • positive 俱뫮잺 pozitivan
    • neutral 俱뫮잺 neutralan
    • negative 俱뫮잺 negativan

    You can evaluate this dataset directly as follows:

    euroeval --model <model-id> --dataset mms-sr

    Named Entity Recognition

    UNER-sr

    This dataset was published in this paper.

    The original dataset consists of 3,328 / 536 / 520 samples for the training, validation, and test splits, respectively. We use 1,024 / 256 / 2,048 samples for our training, validation and test splits, respectively. The train and validation splits are subsets of the original splits, while the test split is created using additional samples from the train and validation splits.

    Linguistic Acceptability

    ScaLA-sr

    This dataset was published in this paper and was automatically created from the Serbian Universal Dependencies treebank by assuming that the documents in the treebank are correct, and corrupting the samples to create grammatically incorrect samples. The corruptions were done by either removing a word from a sentence, or by swapping two neighbouring words in a sentence. To ensure that this does indeed break the grammaticality of the sentence, a set of rules were used on the part-of-speech tags of the words in the sentence.

    The original full dataset consists of 1,024 / 256 / 2,048 samples for training, validation and testing, respectively (so 3,328 samples used in total). These splits are used as-is in the framework.

    Here are a few examples from the training split:

    {
    "text": "Hrvatski ministar odbrane Branko Vukeli캖 i njegov srpski kolega Dragan 맛tanovac potpisa캖e u utorak (8. juna) u Zagrebu bilateralni sporazum o saradnji na polju odbrane.",
    "label": "correct"
}

    {
    "text": "콯ene vlasnici i rukovodioci pokazale su veliku upornost u o캜uvanju svojih, posebno tokom ekonomske krize.",
    "label": "incorrect"
}

    {
    "text": "O캜ekuje se da snimanje bude zavr코eno do kraja leta, a monta쬬 bi trebalo da bude gotova do aprila slede캖e godine.",
    "label": "correct"
}

    When evaluating generative models, we use the following setup (see the methodology for more information on how these are used):

    • Number of few-shot examples: 12
    • Prefix prompt:
    U nastavku su re캜enice i da li su gramati캜ki ispravne.
    • Base prompt template:
    Re캜enica: {text}
Gramati캜ki ispravna: {label}
    • Instruction-tuned prompt template:
    Re캜enica: {text}

Odredite da li je re캜enica gramati캜ki ispravna ili ne. Odgovorite sa {labels_str}, i ni코ta drugo.
    • Label mapping:
    • correct 俱뫮잺 da
    • incorrect 俱뫮잺 ne

    You can evaluate this dataset directly as follows:

    euroeval --model <model-id> --dataset scala-sr
